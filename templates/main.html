How RSA Encryption Works

1. Key Generation
  • Two large prime numbers (p and q) are chosen randomly
  • Their product n = p × q is calculated
  • The totient φ(n) = (p-1) × (q-1) is computed
  • A public exponent e is chosen (usually 65537)
  • A private exponent d is calculated as the modular multiplicative inverse of e modulo φ(n)
2. Key Distribution
  • Public Key (e, n) - Share with anyone who wants to send you encrypted messages
  • Private Key (d, n) - Keep this secret! Used to decrypt messages
3. Encryption Process
  • Convert the message to numbers using ASCII values
  • For each number m, calculate c = me mod n
  • The result is the encrypted message (ciphertext)
4. Decryption Process
  • For each number c in the ciphertext
  • Calculate m = cd mod n
  • Convert the numbers back to characters

Application Features

Basic Features
  • Generate RSA key pairs (1024/2048 bits)
  • Encrypt messages using public key
  • Decrypt messages using private key
  • Copy/paste functionality for keys and messages
Advanced Features
  • Share encrypted/decrypted messages via links
  • Download messages and ciphertexts
  • View operation history
  • Secure user authentication

Security Best Practices

  • Never share your private key with anyone
  • Use at least 2048-bit keys for better security
  • Store your keys securely
  • Always verify the sender's identity when receiving public keys
  • Don't reuse the same key pair for different purposes
